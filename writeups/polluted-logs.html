CTF Challenge · Web

Polluted Logs

Challenge design — ANU CTF
Prototype Pollution Node.js Express
Vulnerable Merge Function → Prototype Pollution → Property Injection → RCE / Auth Bypass
01

Problem

The goal was a web challenge that taught JavaScript prototype pollution without leaning on a well-known public CTF template, so competitors had to actually understand the mechanism rather than pattern-match a known writeup.

Why this matteredMost public prototype-pollution challenges reuse the same one or two vulnerable merge libraries. Building a custom vulnerable endpoint forces solvers to read the merge logic themselves instead of recognizing a signature.
02

Design

The app exposes a small Express service with a user-settings endpoint that recursively merges a JSON body into a config object without filtering dangerous keys like __proto__ or constructor.prototype.

Why this matteredPlacing the sink in a settings-merge endpoint (rather than an obviously named 'debug' route) keeps the challenge realistic — this is exactly the kind of code found in real config-merging utilities.
03

Implementation

A deliberately unsafe recursive merge function walks nested keys of the request body and assigns them onto the target object with no denylist. Polluting Object.prototype through this path flips an internal flag later read by an authorization check elsewhere in the app.

Why this matteredSplitting the pollution point and the security-sensitive read into two different files was intentional — it means solvers have to trace how prototype properties propagate through the app, not just spot one obviously bad line of code.
04

Result

Solvers who identify the unsafe merge and pollute the right property flip an authorization check downstream, gaining access to an admin-only route that reveals the flag.

Why this matteredTying the win condition to a downstream authorization bypass — instead of direct RCE — kept the challenge accessible at a Medium difficulty while still requiring a real understanding of how polluted prototypes affect unrelated code paths.
