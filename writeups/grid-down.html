CTF Challenge · Web / Infra

Grid Down

Challenge design — ANU CTF, based on CVE-2021-41773
Apache 2.4.49 Path Traversal CGI RCE
Path Traversal → Arbitrary File Read → CGI Misconfig → RCE
01

Problem

CVE-2021-41773 is a path traversal flaw in Apache HTTP Server 2.4.49 caused by an incomplete fix to the path-normalization logic, letting encoded traversal sequences like %2e%2e/ escape the web root. The goal was to package this as a self-contained, story-driven challenge (a fictional power-grid control panel) rather than a bare vulnerable-VM download.

Why this matteredWrapping a real CVE in a themed scenario keeps the challenge grounded in an actual historical vulnerability class solvers may meet again in the field, while giving it a narrative hook that makes the CTF more engaging than a plain 'exploit this CVE' prompt.
02

Design

The vulnerable Apache 2.4.49 instance serves a mock 'grid control' web app. Path traversal alone grants arbitrary file read; on hosts where mod_cgi is enabled, the same traversal flaw escalates to remote code execution by reaching the cgi-bin path.

Why this matteredIncluding both the file-read and the RCE escalation path let the challenge serve two difficulty tiers from one environment — an easier flag reachable via file read, and a harder flag requiring the full RCE chain.
03

Implementation

The environment was containerized with the specific vulnerable Apache version pinned, mod_cgi enabled intentionally, and a themed file structure (fake 'grid schematics' and credentials) planted to reward the file-read stage before the RCE stage.

Why this matteredPinning the exact vulnerable version rather than 'latest' matters here — later Apache patches closed this exact traversal path, so version drift would silently break the challenge.
04

Result

Solvers who reach the file-read stage recover the first flag from the planted schematics files; solvers who chain through to CGI execution recover a second, harder flag from a root-only file.

Why this matteredLayering two flags behind one vulnerability class rewarded partial progress instead of an all-or-nothing solve, which kept less experienced competitors engaged instead of scoring zero.
