Lightweight Antivirus Scanner

Python Malware Scanner
SHA-256 Signature Detection Heuristics
Directory Walk → Hash Match → Heuristic Check → Alert Log
01

Problem

Most introductory malware-scanner projects stop at a hardcoded hash list. The goal here was to build something closer to how a real lightweight scanner behaves — recursive, with both signature and heuristic detection, and actual logging.

Why this matteredBuilding the defensive side of tooling, even a simple version, sharpens intuition for what an offensive payload needs to evade — a useful complement to the exploitation-focused work.
02

Design

The scanner recursively walks a target directory tree, computing a SHA-256 hash for each file and comparing it against a maintained list of known-malicious hashes.

Why this matteredSHA-256 was chosen over faster, weaker hashes because signature-based detection is only as trustworthy as its collision resistance — a scanner is worthless if hashes can be trivially forged to evade it.
03

Implementation

Alongside exact hash matches, simple heuristic checks flag suspicious characteristics that don't rely on a known signature, catching some variants that a pure hash list would miss.

Why this matteredSignature-only detection fails against anything even slightly modified from a known sample — heuristics are what let the scanner catch minor variants without needing an updated hash for every single one.
04

Result

Every scan produces an alert log recording what was flagged and why, giving a reviewable trail rather than a silent pass/fail.

Why this matteredLogging the reasoning behind each flag (hash match vs. heuristic) makes false positives easier to diagnose and tune later.
