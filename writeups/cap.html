01

Reconnaissance

nmap -p- --min-rate 5000 <TARGET_IP>

Found 3 open ports: FTP (21), SSH (22), HTTP (80).

Why this matteredThree-port Linux boxes on HTB are almost always "the web app is the whole story" — worth going straight at the app rather than spending time on FTP/SSH banner-grabbing.
02

Web Enumeration & IDOR

Visited http://<TARGET_IP> — a Security Dashboard running on Gunicorn. Clicked Security Snapshot, which redirected to /data/1. Manually changed the ID to 0:

http://<TARGET_IP>/data/0

This is an IDOR vulnerability — the app doesn't verify ownership of the scan. Downloaded the pcap file from /data/0.

Why this matteredThe app trusted the client-supplied ID completely instead of checking whether the requesting session owned that scan record — the single most common IDOR pattern: sequential IDs plus no authorization check on read.
03

Credential Discovery

Opened the pcap in Wireshark, followed the TCP Stream on the FTP traffic and found credentials in plaintext:

USER nathan
PASS <redacted>
Why this matteredOld, unencrypted protocols like plain FTP are still deployed in the wild, and a captured packet trace is one of the most direct ways credentials leak — no cracking required, just reading the stream.
04

User Flag

SSH'd into the machine using the FTP credentials (password reuse):

ssh nathan@<TARGET_IP>
# password reused from FTP

cat ~/user.txt
User Flag
Why this matteredCredential reuse across services is one of the most reliable lateral-movement assumptions to test on any box — a leaked credential for one protocol is very often valid for another.
05

Privilege Escalation

Checked for binaries with special Linux capabilities:

getcap -r / 2>/dev/null

Found:

/usr/bin/python3.8 = cap_setuid,cap_net_bind_service+eip

cap_setuid allows Python to change its UID to root (0).

Why this matteredLinux capabilities are a separate, more granular privilege model than the classic SUID bit, and they're easy to miss if enumeration only checks find / -perm -4000. getcap -r / is the equivalent sweep for capabilities and deserves to be a standard step, not an afterthought.
06

Root Flag

Exploited the capability to spawn a root shell:

python3.8 -c 'import os; os.setuid(0); os.system("/bin/bash")'

cat /root/root.txt
